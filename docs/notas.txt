remainder string
│
├─ simples
├─ pouca memória estrutural
├─ fácil debugging
└─ pode gerar O(n²)

linked list
│
├─ incremental
├─ streaming friendly
├─ append barato
└─ traversal pior

vector/dynamic array
│
├─ cache friendly
├─ rápido
├─ amortized O(n)
└─ realloc ocasional

ring buffer
│
├─ altíssima performance
├─ reutiliza memória
└─ lógica mais complexa

rope/tree
│
├─ gigantesco volume
├─ inserções rápidas
└─ complexidade alta

hashmap fd->state
│
├─ múltiplos streams
├─ escalável
└─ gerenciamento extra
